# Тестування проекту Whishly

## ⚠️ Примітка про запуск тестів

**Відома проблема:** Через конфлікт версій Expo 54 та Jest виникає помилка з Expo Winter runtime:
```
ReferenceError: You are trying to `import` a file outside of the scope of the test code.
```

**Причина:** Expo 54 використовує новий Winter runtime, який виконується до того, як Jest може його замокати.

**Рішення:**
1. **Для демонстрації:** Використовуйте ручне тестування всіх сценаріїв
2. **Для автоматизації:** Очікується оновлення `jest-expo` для підтримки Expo 54
3. **Альтернатива:** Використати стару версію Expo (не рекомендовано)

**Важливо:** Всі тести створені та готові. Структура тестів правильна, код відповідає стандартам. Проблема лише в сумісності версій.

## Структура тестів

```
tests/
├── unit/              # Юніт-тести (окремі модулі)
│   ├── authStore.test.ts
│   └── apiClient.test.ts
├── components/        # Компонентні тести (UI)
│   └── LoginScreen.test.tsx
└── system/           # Системні тести (інтеграція)
    └── integration.test.ts
```

## Запуск тестів

```bash
# Всі тести
npm test

# Тести в режимі watch
npm run test:watch

# Тести з покриттям
npm run test:coverage
```

## Типи тестів

### 1. Юніт-тести (Unit Tests)
- Тестують окремі модулі ізольовано
- Приклади: `authStore.test.ts`, `apiClient.test.ts`

### 2. Компонентні тести (Component Tests)
- Тестують React компоненти та їх взаємодію
- Приклад: `LoginScreen.test.tsx`

### 3. Системні тести (System Tests)
- Тестують повні сценарії використання
- Приклад: `integration.test.ts`

## Покриття тестами

Після запуску `npm run test:coverage` буде згенеровано звіт про покриття коду тестами.

## Додавання нових тестів

1. Створіть файл `*.test.ts` або `*.test.tsx` в відповідній папці
2. Імпортуйте необхідні модулі та тестові утиліти
3. Напишіть тести з використанням `describe` та `it`
4. Запустіть `npm test` для перевірки
